Safari で Push をデバッグする

MeshCore 等のドキュメントで言う「.local 拒否」は PWA の URL（https://xxx.local）ではなく、 サーバーが JWT に入れる VAPID sub （例: mailto:noreply@meshcore.local）の話です。 403 BadJwtToken になります。201 が出ていれば sub は通過済みです。

ページの Web インスペクタ → コンソール:

navigator.serviceWorker.ready
  .then(r => r.pushManager.getSubscription())
  .then(s => console.log(s?.endpoint?.slice(-24)))

ブレークポイントがヒットしない場合、別 endpoint への送信か active 以外の SW をデバッグしている可能性があります。 診断情報の endpoint 末尾（…xxxx）が Mac 側と一致するか確認してください。

  1. Safari メニュー → 設定 → 詳細 → 「メニューバーに"開発"メニューを表示」
  2. 開発 → Service Workers → このサイトの sw.js → インスペクト
  3. テスト送信の前に SW 用インスペクタを開いておく
  4. または URL に ?debugPush=1 を付けて開く（push 受信時に debugger で停止）
  5. SW コンソールに [Insta360Sync SW] push event received が出るか確認
  6. システム設定 → 通知 → Safari で通知が許可されているか確認

ルート証明書

Mac が生成した自己署名ルート証明書を端末にインストールすると、ブラウザや PWA が この Mac の HTTPS を「信頼済み」として扱えるようになります。一度信頼設定を有効に すれば、以降このセクションを開く必要はありません。

インストール手順

iOS / iPadOS

  1. Safari で「証明書をダウンロード」ボタンをタップし、プロファイルをダウンロードします。
  2. 設定 → 一般 → VPN とデバイス管理 → ダウンロードされたプロファイル を開き、インストールします。
  3. 設定 → 一般 → 情報 → 証明書信頼設定 で当該証明書のスイッチをオンに します（フル信頼の有効化）。
  4. Safari を再読み込みし、鍵アイコンが警告なしになれば完了です。

Android

  1. Chrome で「証明書をダウンロード」ボタンをタップしてファイルを保存します。
  2. 設定 → セキュリティとプライバシー → その他 → 暗号化と認証情報 → 証明書のインストール → CA 証明書 を選択します（機種により表記が異なる場合があります）。
  3. ダウンロードしたファイルを選択し「ユーザーによりインストールされた CA 証明書」 として保存します。
  4. Chrome を再起動すると信頼されます（一部のアプリはユーザー CA を信頼しない ことがあります）。

フィンガープリント (SHA-256) が Mac 設定画面と一致していることを確認してから インストールしてください。
